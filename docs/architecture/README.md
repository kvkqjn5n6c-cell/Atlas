# Architecture Atlas

Ce dossier formalise l'architecture cible d'Atlas après la Phase 31.

Atlas est défini comme un copilote décisionnel métier capable de transformer les données opérationnelles, les KPI, les documents et les connaissances de l'entreprise en recommandations concrètes et explicables.

## Documents

- [ATLAS_CORE.md](ATLAS_CORE.md) : socle applicatif.
- [ATLAS_MEMORY.md](ATLAS_MEMORY.md) : mémoire structurée par organisation.
- [ATLAS_BUSINESS_ENGINE.md](ATLAS_BUSINESS_ENGINE.md) : moteur métier.
- [ATLAS_CONNECTORS.md](ATLAS_CONNECTORS.md) : connecteurs et ingestion.
- [ATLAS_AGENTS.md](ATLAS_AGENTS.md) : agents spécialisés.
- [ATLAS_AI_LAYER.md](ATLAS_AI_LAYER.md) : couche IA générative encadrée.
- [ATLAS_SECURITY.md](ATLAS_SECURITY.md) : sécurité, audit, RGPD.
- [ATLAS_MULTITENANT.md](ATLAS_MULTITENANT.md) : isolation multi-organisation.
- [ATLAS_ROADMAP.md](ATLAS_ROADMAP.md) : trajectoire cible.

## Principe directeur

Le moteur métier, la mémoire et les intégrations doivent rester exploitables sans IA générative. L'IA intervient ensuite comme accélérateur de synthèse, d'explication et de recommandation, toujours avec des garde-fous.
